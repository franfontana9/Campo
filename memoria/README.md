# Memoria del proyecto Campo

Esta carpeta guarda toda la información clave del proyecto Campo (marketplace de granos en Argentina): conversaciones, decisiones, arquitectura, esquemas y planes. Sirve como memoria compartida entre el equipo humano y el asistente IA.

## Estructura

- `product-definition.md` — qué es Campo, a quién sirve, qué incluye y qué no el MVP.
- `arquitectura.md` — stack, patrones, decisiones técnicas.
- `schema-db.md` — tablas, relaciones, RLS.
- `pantallas.md` — inventario de rutas y qué hace cada una.
- `plan-implementacion.md` — sprints y prioridades.
- `conversaciones/` — resúmenes de sesiones de trabajo (una por día o por tema).
- `decisiones.md` — registro de decisiones con fecha y racional.

## Reglas

- Cuando cambie algo estructural (stack, schema, rutas), actualizar el doc correspondiente en vez de dejarlo desactualizado.
- Las conversaciones se guardan como resumen, no transcripción completa.
- Si una decisión se revierte, no borrar: agregar entrada nueva marcando el cambio.
